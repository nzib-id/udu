## Decisions locked

### Product direction
- **Name:** Udu (ikutin domain)
- **Genre:** Tribal survival observation sim, 1 karakter MVP
- **Player role:** Pure observer — no intervention, no control
- **Growth path:** MVP (1 char) → v2 tribe → v3+ oral tradition

### Tech stack
- **Frontend:** Phaser 3 + TypeScript (Vite bundler)
- **Backend:** Node.js 22 + TypeScript
- **DB:** SQLite (better-sqlite3)
- **Transport:** WebSocket (state push from server)
- **LLM:** Ollama qwen3:8b lokal di Windows host (172.21.160.1:11434)
- **Docs:** VitePress
- **Host:** pm2 + cloudflared tunnel on Nzib's local server
- **Domains:** udu.loodee.art (game, public), docs.udu.loodee.art (docs, public)

### AI architecture
- **Two-layer brain:**
  - Utility AI: real-time action scoring (1 Hz decision tick)
  - LLM Reflection Engine: tiap game-day (24 min real) call Ollama, parse rules JSON, merge ke utility weights
- **Spirit memory:** Rules persist across deaths. New character inherits rules dari lineage.

### Gameplay
- **Map:** 80x60 tiles logical (16px), camera zoom 2-3x, responsive
- **Time:** 1 min real = 1 game-hour, 24 min = 1 game-day
- **Stats:** hunger, thirst, bladder, energy (4 core)
- **Decay:** hunger -2/hr, thirst -4/hr, bladder +3/hr, energy -3/hr
- **Resources:** forage (bush berry, tree fruit, river water) + hunting (ayam hutan, ikan sungai)
- **Tools:** kayu, NO durability
- **Cook:** hybrid — raw (less hunger, sickness) vs cooked (safe, full)
- **Death:** 6 game-hours at hunger/thirst=0 → die, respawn with inherited rules
- **Palette:** 4-phase time-of-day cycle (dawn/day/dusk/night)

### Scope exclusions (MVP)
- Player interaction / control
- Predators / combat risk
- Weather / seasons
- Multi-character / tribe
- Complex pathfinding beyond grid A*
- Graphics fidelity beyond 16x16 pixel art
- Temperature / sickness-as-critical-stat
