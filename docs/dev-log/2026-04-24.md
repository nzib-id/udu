## Mobile zoom + HUD toggle

Nzib asked for pinch-zoom (bounded by map edges) and a show/hide button for the HUD/stats panel.

**Pinch zoom (`MapScene.ts`):** added `this.input.addPointer(1)` — Phaser's default max-active-pointers is 1, so pinch needed a second slot. Two-finger pinch tracks distance delta between `pointer1` + `pointer2`, zooms proportionally via new `zoomAround(screenX, screenY, nextZoom)` that anchors the world point under the pinch midpoint while scaling. Wheel zoom re-routes through the same helper. Single-finger drag is disabled the moment a second pointer enters, then stays off until both lift (prevents drag-induced scroll jumps as the second finger touches down).

**Pan clamp (`clampScroll()`):** `setBounds(0, 0, worldW, worldH)` clamps Phaser's own scroll helpers but our manual `cam.scrollX/Y` writes (follow-mode lerp, drag delta, pinch re-anchor) bypass that. New helper re-clamps scroll to `[0, world - viewport/zoom]` after every manual write, plus on resize and fitCamera init.

**HUD toggle (`index.html`):** new `<button id="hud-toggle">` fixed top-right (36×36, `rgba(0,0,0,0.55)` bg matching HUD), inline script toggles `.hidden` on `#hud` (CSS `display:none`). Button glyph flips `☰` ↔ `▤` based on state. Persisted to `localStorage['udu-hud-hidden']`. Default visible on all devices.

## Responsive camera pass (real-device iPhone fix)

Nzib tested the pinch-zoom build on a real iPhone and got a black bar at the bottom of the screen plus asymmetric scroll (could drag down past world bounds, couldn't drag up after zoom). Headless CDP screenshots looked fine — real Safari rendered differently.

**Design decision:** switched from "cover fit floor" (`Math.max`) to hybrid — canvas fills 100% viewport, default zoom computed so the **shorter** axis shows ~15 tiles (always wider than cover floor on any aspect ratio), camera follows character by default, drag breaks follow, on-screen tombol `＋` / `−` / `◎` bottom-left for zoom + recenter. Industry pattern: Don't Starve–style fluid camera over Stardew-style fixed design resolution.

**Root cause of the black bar (the invisible one):** Phaser camera default `originX/Y = 0.5`. With that origin, the rendered world position is offset by `(cam.size - displayedSize) / 2`. On portrait phones the zoom is tall-axis-limited → displayed height < canvas height → the world gets pushed up by half the difference, leaving a big black band below. CDP tests passed because the view rect reported correctly; the **rendered pixels** were shifted. Fix: `this.cameras.main.setOrigin(0, 0)` in `MapScene.create()` right after `TerrainLayer`. Scroll math (`scrollX/Y = world coord at top-left of viewport`) now matches mental model used by `clampScroll()` / follow.

**Other changes (`MapScene.ts`):**
- `followCharacter` default → `true` (was `false`).
- New `DEFAULT_TILES_VIEW = 15` + `defaultZoom()` helper: `min(cam.w, cam.h) / (15 * tileSize)`, floored to `fitZoom()`.
- `fitCamera()` now uses `defaultZoom` + centers on character (if spawned) else world midpoint.
- `clampScroll()` rewritten to center-letterbox when `viewW >= worldW` (or Y) instead of pinning to 0.
- `scale.resize` handler preserves zoom when not following; re-fits when following; clamps.
- `recenterOnCharacter()` public: re-enables follow + re-fits.
- `spawnCharacter()` snaps camera on first spawn.
- `game.events.on('recenter')` lets HTML button ping scene without coupling.

**Camera control UI (`index.html`):** 3 buttons in `<div id="camera-controls">` bottom-left (44×44 dark, `rgba(0,0,0,0.55)` matching HUD): `cam-zoom-in`, `cam-zoom-out`, `cam-recenter`. Wired via inline script that calls `scene.zoomAround(cam.w/2, cam.h/2, next)` for zoom buttons and `scene.recenterOnCharacter()` for the focus button.
