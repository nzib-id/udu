# Dev Log

Catatan harian tiap session development — konteks, yang dikerjain, decisions, blockers.

## Entries

- [2026-04-23 — Planning & Docs Init](./2026-04-23)

## How to use

Tiap session kerja Udu, Loodee bikin (atau update) dev-log entry hari itu. Content:

1. **Session summary** — apa goal session hari ini
2. **Decisions** — choice yang di-make (dengan reasoning)
3. **What was built** — file changes, features implemented
4. **Blockers / open questions** — hal yang belum diresolusi
5. **Next session** — apa yang harus dikerjain lanjutan

Purpose: future Loodee session bisa baca log terakhir, langsung nyambung tanpa brief ulang.
